Section 1
Introduction & Purpose
The Payroll Management System (PMS) is a comprehensive, end-to-end B2B solution built to automate and manage the complete payroll lifecycle for organisations.
🏢

Organisation & Configuration

Company hierarchy, pay schedules, work locations, departments, and statutory rules configuration.

👤

Employee Onboarding & CTC

Full employee profile, versioned CTC structures, salary components (earnings, benefits, reimbursements).

📜

Statutory & Tax Compliance

EPF, ESI, Professional Tax, TDS dynamic resolution, and Statutory Bonus per Indian labour law.

⚙

Pay Run Execution

Core payroll engine — validate employees, compute gross, apply deductions, generate net pay.

🎁

Bonus Management

Statutory bonus, custom one-time payouts, two-part splits, 6th/12th month flat bonuses.

🌐

Employee Self-Service Portal

Secure payslip access, salary details, IT declarations, and proof of investment submissions.

📋

Tax Declarations & IT

Dynamic TDS spread across fiscal months, POI upload/approval, compliance report export.

📄

Payslip Generation

Auto-generate PDFs, upload to Cloudinary, dispatch via Brevo email, portal download.

Section 10
Non-Functional Requirements
Quality attributes, performance benchmarks, and security standards the system must meet.
⚡ Software Quality Attributes
AttributeRequirement
AvailabilityMinimum 99.5% uptime during payroll processing windows (normal working hours)
CorrectnessEPF, ESI, PT, TDS, Bonus computations must produce legally compliant and mathematically accurate results
PerformancePay run generation for ≤1,000 employees must complete within 60 seconds. API responses under 2 seconds for standard operations
FlexibilityConfigurable statutory rules (EPF wage caps, PT slabs, bonus rates) without code changes. New states/components addable without downtime
IntegrityReferential integrity across all tables. Backward-compatible schema changes ensure no data corruption on updates
MaintainabilityAdditive, backward-compatible features. Layered architecture (Entity → DTO → Mapper → Service → Controller) for easy maintenance
PortabilitySpring Boot (Linux/Windows). React (Chrome, Firefox, Edge). Azure/AWS/on-premise deployment ready
ReliabilityIdempotent pay processing — re-runs for same period don't duplicate records. Retry logic for Brevo/Cloudinary failures
SecurityAll endpoints protected by JWT via Keycloak. HTTPS/TLS 1.2+ in transit. Sensitive data (PAN, bank, UAN) protected at rest. RBAC at backend level
TestabilityUnit tests for all statutory logic. Integration tests for full pay run flow. Property-based tests for mapper symmetry
UsabilityStep-by-step HR portal UI with validation feedback. Employee portal is mobile-responsive and intuitive for non-technical users
ScalabilitySupports 10 to 10,000 employees. DB indexing + pagination for large volumes. Horizontal scaling via cloud deployment
Security

🔒 Data Encryption

All API communications over HTTPS (TLS 1.2/1.3). PAN, bank account, UAN stored with access controls. Cloudinary documents served via secure, expiring URLs.

Audit

📝 Audit Logging

All payroll processing, CTC revisions, and statutory config changes logged with timestamps and user IDs. Pay run lifecycle events tracked with full status history.

API

🛡 API Security

Every REST API endpoint requires valid JWT in Authorization header. orgId validated against authenticated user roles. Public endpoints are rate-limited.

Tokens

⏱ Token Management

Portal invitation tokens expire in 5 minutes. Expired links show styled session-expired page. Re-invite generates fresh token and re-enables Keycloak user.
